# fra-7w3.2 Queue Runner Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a production-safe P5.2 runner skeleton that accepts agent queue messages, prevents duplicate run side effects, enforces per-agent concurrency, and runs an injectable agent loop transactionally.

**Architecture:** Extend `services/agents` with focused modules: `agent-run-repo.ts` owns run lifecycle rows in `agent_run_logs`, `queue-runner.ts` owns message validation/idempotency/concurrency decisions, and `agent-loop.ts` owns ordered stage orchestration with injectable stage functions. Existing `watermarks.ts` remains the transaction boundary for finding/activity side effects plus watermark advancement.

**Tech Stack:** TypeScript on Node `--experimental-strip-types`, `pg`-style `QueryExecutor`, existing `services/agents` package, existing `agent_run_logs`, `run_activities`, `findings`, and `agents.watermarks` tables.
